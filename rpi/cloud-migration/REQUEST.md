# VPS/クラウド移行

## 共通コンテキスト

- **プロジェクト**: Discord Minutes Bot — Discord音声チャンネルの録音から自動で議事録を生成するBot
- **技術スタック**: Python 3.10+, discord.py 2.3+, faster-whisper (large-v3, CUDA), Claude API, Google Drive API
- **設計原則**: Pipeline-first, async by default, graceful degradation, multi-guild support, minimal state
- **用語集**: Docker: コンテナ化プラットフォーム, CUDA: NVIDIA GPU計算フレームワーク, systemd: Linuxサービス管理

## この機能について

### フェーズ・優先度
- **企画書フェーズ**: 今後の拡張候補（スコープ外）
- **実装順序**: Ext-8

### 概要
ローカルPC運用からVPS/クラウドサーバーへ移行し、PC常時稼働の制約を解消する。Docker対応は既に完了しているため、GPU付きクラウドインスタンスへのデプロイ手順とインフラ設定を整備する。

### 元企画書からの該当箇所
> VPS / クラウドへの移行（PC常時稼働が難しくなった場合）

## 受入基準
1. GPU付きクラウドインスタンスでDockerコンテナが正常起動する
2. Craig録音→議事録生成→Discord投稿のフルパイプラインが動作する
3. インスタンス再起動後にBotが自動復帰する
4. デプロイ手順書に従って新環境をゼロからセットアップできる

## 備考
- Docker対応（Phase 6）が完了しているため、技術的にはGPU付きインスタンスにデプロイするだけ
- 主な作業はインフラ設定とドキュメント整備
- GPU VPS候補: Vast.ai, Lambda Cloud, RunPod, AWS EC2 (g4dn), GCP (T4)
- Whisper large-v3のVRAM要件（6GB+）に合うインスタンスタイプの選定が必要
- OPEN-03（Windows/Mac自動起動）はクラウド移行で不要になる
