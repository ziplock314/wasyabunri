# 詳細議事録フォーマット + トランスクリプト添付

## 概要

Discord投稿時に添付するファイルを2つに増やし、Geminiメモ風の詳細な議事録フォーマットを導入する。

## 現状

- Discord Embed（サマリー）+ 議事録 `.md` ファイル1つを添付
- 議事録は「要約 / アジェンダ / 議論の詳細 / 決定事項 / TODO / 懸念事項」の6セクション構成
- トランスクリプト（文字起こし）はパイプライン内部で生成されるが、ユーザーには公開されていない

## 要件

### 1. 添付ファイルを2つにする

**ファイル1: 議事録（まとめ + 詳細）**
- Google Meet Geminiメモ風の構造化された議事録
- 「まとめ」セクション: 3〜5行の概要 + トピックごとの太字サマリー + 短い説明
- 「詳細」セクション: トピックごとの箇条書き。各項目にタイムスタンプ参照あり、太字の要点 + 詳細な説明文
- 「推奨される次のステップ」セクション: チェックリスト形式のアクションアイテム（担当者明記）

**ファイル2: 文字起こし（トランスクリプト）**
- タイムスタンプ付き話者ラベル付きの生テキスト
- `### HH:MM:SS` のセクション区切り + `**話者名:** テキスト` のフォーマット
- merger.py で生成済みのトランスクリプトをMarkdown形式に整形して添付

### 2. プロンプトテンプレートの更新

- `prompts/minutes.txt` をGeminiメモの「まとめ + 詳細 + 推奨される次のステップ」形式に書き換え
- 詳細セクションではトランスクリプトのタイムスタンプを参照する形式（例: `([00:04:45])`）
- 情報量はGeminiメモと同等レベル（長い会議では30+項目の詳細）

### 3. max_tokens の引き上げ

- 現在 `4096` → `8192` に引き上げ（詳細な出力に対応）

### 4. Discord Embed（サマリー）は概ね現状維持

- 既存の Embed 表示（参加者、要約、決定事項、話者統計）は大きく変更しない
- poster.py の `_SUMMARY_PATTERN` / `_DECISIONS_PATTERN` を新フォーマットに合わせて調整が必要な場合のみ修正

## 参考資料

- Geminiメモ例: `/mnt/c/Users/shimo/Downloads/TOONIQ取締役Sync - 2026_03_16 14_01 JST - Gemini によるメモ.md`
- Gemini文字起こし例: `/mnt/c/Users/shimo/Downloads/TOONIQ取締役Sync - 2026_03_16 14_01 JST - Gemini によるメモ (1).md`

## 変更対象ファイル（想定）

1. `prompts/minutes.txt` — プロンプトテンプレート書き換え
2. `src/poster.py` — トランスクリプトファイル添付の追加、Embed正規表現の調整
3. `src/pipeline.py` — `transcript` を `post_minutes` に渡す
4. `config.yaml` — `max_tokens` 引き上げ
5. テスト更新
