# Product Requirements: minutes-search

**Feature**: 過去議事録検索
**Date**: 2026-03-17
**Priority**: Ext-4

---

## User Stories

### US-1: 議事録の自動アーカイブ
**As a** 会議参加者
**I want** 生成された議事録が自動的に永続保存される
**So that** 過去の会議内容をいつでも参照できる

**受入基準**:
- パイプラインで議事録が生成されたとき、自動的にアーカイブに保存される
- メタデータ（日付、参加者、ギルドID、チャンネル名）が議事録と共に保存される
- アーカイブ保存の失敗が議事録のDiscord投稿をブロックしない

### US-2: キーワード検索
**As a** 会議参加者
**I want** `/minutes search <keyword>` で過去の議事録を検索できる
**So that** 特定のトピックや決定事項を素早く見つけられる

**受入基準**:
- キーワードを入力すると、マッチする議事録が最大5件表示される
- 日本語キーワードで正しく検索できる
- 検索結果はEmbed形式でスニペット付きで表示される
- 自分のギルドの議事録のみが検索対象になる

### US-3: 検索結果の表示
**As a** 会議参加者
**I want** 検索結果に日付・参加者・マッチ箇所が表示される
**So that** どの会議の内容かすぐに判別できる

**受入基準**:
- 各検索結果に会議日時が表示される
- 各検索結果に参加者名が表示される
- マッチ箇所の前後テキストがスニペットとして表示される
- 関連度順にソートされる

---

## Business Value

### 定量的価値
- 過去議事録の参照時間: 数分（Discord内手動検索）→ 数秒（コマンド一発）
- 議事録の消失防止: 現在は事実上揮発的（キャッシュ1件のみ）→ 全件永続保存

### 定性的価値
- 会議ライフサイクル（capture → publish → **retrieve**）の完成
- 会議回数の増加に比例して価値が増大
- Ext-5（external-export）のアンブロック

### 市場適合
- 全主要競合（Otter.ai, Fireflies, Fellow）が標準提供する機能
- この機能の欠如は明確なプロダクトギャップ

---

## Scope

### In Scope
- SQLite FTS5による全文検索アーカイブ
- `/minutes search <keyword>` スラッシュコマンド
- パイプラインからの自動アーカイブ
- ギルド単位のデータ分離
- Embed形式の検索結果表示

### Out of Scope（将来対応）
- 日付範囲フィルタリング（Nice to Have — Phase 2以降で検討）
- 話者名フィルタリング（Nice to Have — Phase 2以降で検討）
- 元メッセージへのリンク（Nice to Have — message_idカラムは先行設置）
- ページネーション（ボタンナビゲーション — 初回は件数制限のみ）
- 既存キャッシュからのデータ移行（バックフィル）

---

## Non-Functional Requirements

| 項目 | 要件 |
|------|------|
| 応答速度 | 100件のアーカイブで検索応答1秒以内 |
| ストレージ | 1,000件で約9MB（SQLite + FTS5インデックス） |
| セキュリティ | ギルド間データ漏洩禁止（guild_id WHERE句で強制） |
| 可用性 | アーカイブ書込み失敗がパイプラインをブロックしない |
| 互換性 | 既存 `minutes_cache.json` は維持（dedup用途は別） |
