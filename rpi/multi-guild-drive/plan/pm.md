# Product Requirements: マルチギルド対応（Google Drive監視 + エラーロール）

## 概要

既にコミットされたマルチギルドアーキテクチャの完成。Google Drive監視とエラー通知ロールがギルドごとに正しく機能するようにする。

## ビジネス価値

- **正確性の修正**: 2つ目以降のギルドでDrive経由の議事録がサイレントに投稿されないバグを解消
- **運用改善**: ギルド固有のエラーロールでメンションが正しく機能
- **拡張性**: マルチサーバー運用のブロッカーを解消

## ユーザーストーリー

### US-1: ギルド別Drive監視（Must）
**As a** マルチサーバー管理者
**I want to** 各ギルドに固有のDriveフォルダIDを設定
**So that** 検出されたファイルがそのギルドの出力チャンネルに投稿される

**受入基準**:
- [ ] ギルドごとに `google_drive.folder_id` を設定可能
- [ ] 検出ファイルは設定されたギルドの `output_channel_id` に投稿される
- [ ] 複数ギルドの DriveWatcher が並行稼働する
- [ ] ギルドごとに Drive 監視の有効/無効を切り替え可能

### US-2: ギルド別エラーロール（Must）
**As a** マルチサーバー管理者
**I want to** `error_mention_role_id` をギルドごとに設定
**So that** エラー時にそのサーバーの管理者ロールが正しくメンションされる

**受入基準**:
- [ ] ギルドごとに `error_mention_role_id` を設定可能
- [ ] 未設定の場合はグローバル設定にフォールバック
- [ ] pipeline.py の4箇所全てでギルド別ロールIDが使用される

### US-3: 後方互換（Must）
**As a** 単一サーバー運用者
**I want to** 既存の config.yaml を変更せずに動作
**So that** アップデートで運用が壊れない

**受入基準**:
- [ ] グローバル `google_drive` セクションの設定が引き続き動作
- [ ] グローバル `error_mention_role_id` がフォールバックとして機能
- [ ] ギルド別設定がない場合のデフォルト動作が現在と同一

### US-4: ギルド別Drive監視ステータス（Nice-to-have）
**As a** サーバー管理者
**I want to** `/minutes drive-status` でそのギルドのDrive監視状態を確認
**So that** 監視が正しく動作しているか把握できる

**受入基準**:
- [ ] コマンド実行ギルドのDrive watcher状態を表示
- [ ] 全ギルドの概要も表示（管理用）

## スコープ外

- 新規外部依存の追加
- DriveWatcher内部ロジックの変更
- Craig検知フロー（既にマルチギルド対応済み）
- スラッシュコマンドのスコープ変更（既にギルドスコープ）
- StateStore / MinutesArchive の変更（既にギルド対応済み）
- 新しいスラッシュコマンドの追加

## 優先度

**P1** — 正確性の修正は機能追加に優先する。現状はサイレント障害であり、マルチギルド運用者のブロッカー。
