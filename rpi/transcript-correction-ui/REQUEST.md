# 文字起こし手動修正UI

## 共通コンテキスト

- **プロジェクト**: Discord Minutes Bot — Discord音声チャンネルの録音から自動で議事録を生成するBot
- **技術スタック**: Python 3.10+, discord.py 2.3+, faster-whisper (large-v3, CUDA), Claude API, Google Drive API
- **設計原則**: Pipeline-first, async by default, graceful degradation, multi-guild support, minimal state
- **用語集**: 統合トランスクリプト: 話者別文字起こしを時系列にマージしたテキスト, Segment: 文字起こしの最小単位（開始時間、終了時間、テキスト、話者）

## この機能について

### フェーズ・優先度
- **企画書フェーズ**: 今後の拡張候補（スコープ外）
- **実装順序**: Ext-7

### 概要
文字起こし結果をブラウザ上で手動修正できるWebインターフェースを提供する。Whisperの文字起こし精度が不十分な場合（専門用語、固有名詞等）に、議事録生成前にトランスクリプトを修正できるワークフローを実現する。

### 元企画書からの該当箇所
> 文字起こし結果の手動修正UI（Web画面）

### 要件

#### Must Have（実装必須）

- セグメントの分割・統合 [←R-79]

### UI/UX
- Web画面（ローカルHTTPサーバー or Discord Botからリンク提供）
- 左パネル: セグメントリスト（話者、タイムスタンプ、テキスト）
- 右パネル: 編集エリア
- 「修正完了・議事録生成」ボタンでパイプライン続行

### データ
- トランスクリプトデータ: JSON形式（Segment配列）
- 修正済みトランスクリプト: 同形式で保存
- 音声ファイル: ストリーミング再生用にローカル保持

### API
- ローカルHTTPサーバー（FastAPI or aiohttp）
  - `GET /transcripts/{id}` — トランスクリプト取得
  - `PUT /transcripts/{id}` — 修正済みトランスクリプト保存
  - `POST /transcripts/{id}/generate` — 議事録生成開始

### 非機能要件
- Webサーバーはオプション（config.yamlで有効/無効）
- セキュリティ: ローカルネットワークのみ、認証トークン付き
- 「minimal state」原則との折り合い: 修正データは一時的、生成後に削除

### 適用される共通制約
- 技術スタック: Python標準ライブラリ or 軽量フレームワーク（FastAPI推奨）
- セキュリティ: Web UIへのアクセス制限（ローカルのみ or トークン認証）

## 対象コンポーネント
- `src/web/` — 新規ディレクトリ: Web UIサーバー
- `src/web/server.py` — HTTPサーバー + APIエンドポイント
- `src/web/static/` — HTML/CSS/JS（フロントエンド）
- `src/pipeline.py` — 修正済みトランスクリプトからのパイプライン再開
- `src/config.py` — WebConfig データクラス追加
- `config.yaml` — web セクション追加
- `tests/test_web_server.py` — APIテスト

- **先行（この機能の前に必要）**: なし（ただし規模が大きいため他の機能完了後を推奨）
- **後続（この機能が完了すると着手可能）**: なし

## 受入基準
1. ブラウザでトランスクリプトのセグメント一覧が表示される
2. セグメントのテキストを編集して保存できる
3. 修正後のトランスクリプトから議事録が正しく生成される
4. Web UIが無効の場合、従来のパイプラインが正常動作する

## 備考
- **規模XL**: Webフロントエンド開発が必要で、現在のプロジェクト（Python CLIベース）とは技術領域が異なる
- Discordのモーダル/フォーム機能で簡易版を実現する方法もある（Research段階で比較検討）
- 音声同期再生はWeb Audio APIまたはHTML5 audioタグで実現可能
- 「minimal state」原則に従い、Web UIはオプショナルなアドオンとして設計
